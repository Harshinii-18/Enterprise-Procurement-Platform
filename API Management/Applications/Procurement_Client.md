# Procurement Client Application

## Overview

**Procurement_Client** is the consumer application registered in SAP API Management for the Enterprise Procurement Platform.

The application subscribes to the **Enterprise_Procurement_APIs** product and represents an external client consuming the exposed enterprise APIs.

---

## Application Name

`Procurement_Client`

---

## Associated API Product

`Enterprise_Procurement_APIs`

---

## Authentication

The application is configured to use:

- API Key
- OAuth 2.0 Client Credentials

> **Note:** API Key authentication is used for the primary demonstrations in this project. OAuth 2.0 Client Credentials was implemented and validated within the SAP API Management trial environment.

---

## Subscription Status

**Approved**

---

## Generated Credentials

The following credentials are generated by SAP API Management during application registration:

- Client ID 
- Client Secret 
- API Key 

No credentials or access tokens are committed to this repository for security reasons.

---

## Usage

After subscribing to the API Product, the client application can:

- Generate an OAuth 2.0 access token using the Client Credentials grant (if enabled).
- Invoke the enterprise APIs by including the required authentication credentials.
- Consume the Procurement, Supplier, and Employee API proxies exposed through SAP API Management.
